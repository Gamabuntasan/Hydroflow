Atención: Has abierto la página desde el sistema de archivos (file://). Firebase Authentication normalmente no funciona desde file:// por motivos de seguridad.
Opciones sencillas para que funcione sin usar la terminal:
  1. Subir la carpeta del proyecto a un servicio de hosting estático (sin CLI) como Netlify (arrastrar y soltar): app.netlify.com/drop.
  2. Usar Firebase Hosting (requiere un despliegue inicial con Firebase CLI) o añadir tu dominio de hosting resultante en la consola de Firebase → Authentication → Authorized domains.
  3. Si quieres seguir en local sin subir, ejecuta un servidor estático (por ejemplo Live Server en VS Code); es rápido y evita problemas de permisos.
Si quieres, puedo guiarte paso a paso para subir a Netlify (no necesitas usar la terminal) o desplegar en Firebase Hosting.
